# storage

캐시, 로컬 파일, 인덱스, 누적 리포트 데이터처럼 저장소 접근과 파일 경로 정책을
담당한다.

## 여기에 둔다

- output directory 내부 경로 규칙
- cache 파일 읽기/쓰기 공통 유틸리티
- 누적 리포트/메타 데이터 접근 계층
- index 파일 갱신
- storage migration 보조 로직

## 여기에 두지 않는다

- 외부 source 수집
- LLM provider 실행
- stage별 데이터 처리 정책
- 사람이 읽는 리포트 문장 생성

## 판단 기준

파일 시스템이나 저장소에 어떻게 접근할지, 어디에 저장할지를 결정하는 코드는
storage에 둔다. 특정 도메인 모델 자체는 contracts나 해당 도메인 패키지에 둔다.
