# 🧭 DB 설계 가이드

## 1. 설계 원칙

- 단일 소스는 `schema.dbml`로 유지
- 테이블/컬럼 이름은 `snake_case`
- PK/FK/UNIQUE/NOT NULL은 근거가 있을 때 명시
- `created_at`, `updated_at` 기본 포함

## 3. 데이터 수집 절차 반영

1. Raw 수집 저장
2. 정제/중복 제거
3. 도메인 테이블 반영
4. 실패/재시도 기록

## 4. 확장성/성능 전략

### 4.1 확장성
- 대용량 컬럼/테이블 후보
- 파티셔닝 후보 컬럼
- 아카이빙 기준

### 4.2 조회 성능
- 주요 조회 API별 인덱스 매핑
- 복합 인덱스 전략
- 정렬/필터 컬럼 점검
