description: 이슈 컨벤션에 맞는 제목과 본문을 자동 생성하고 gh CLI로 GitHub 이슈를 생성한다. Todo 체크리스트 형태의 구현 계획을 포함한다.
---

## 동작 순서

### 1. 구현 내용 파악
사용자가 인자로 구현할 기능 설명을 전달한 경우 그대로 사용한다.

- 인자가 없으면 구현할 기능에 대해 입력받는다:
  > 어떤 기능을 구현할 예정인가요?

### 2. 이슈 TYPE 결정
기능 설명을 분석하여 아래 기준으로 `TYPE`을 결정한다:

| TYPE | 기준 |
|------|------|
| `FEAT` | 새로운 기능 추가 |
| `FIX` | 버그 수정 |
| `DOCS` | 문서 작업 |
| `REFACTOR` | 코드 구조 개선 (로직 변경 없음) |
| `REMOVE` | 파일 또는 코드 제거 |
| `CHORE` | 빌드 설정, 패키지, 환경 설정 |
| `TEST` | 테스트 코드 |
| `ENHANCE` | 기존 기능 성능 개선 |

### 3. 이슈 제목 생성
아래 형식으로 제목을 생성한다:

```
[TYPE] 한글 제목
```

- `TYPE`: 영어 대문자
- `제목`: 한글로 작성, 마침표 없음, 간결하게

### 4. 이슈 본문 생성
기능 설명을 분석하여 구현 계획을 Todo 체크리스트 형태로 작성한다:

```markdown
## Todo
- [ ] 구현 항목 1
- [ ] 구현 항목 2
- [ ] 구현 항목 3
```

- 구현 항목은 실제로 작업해야 할 단위로 구체적으로 작성한다.
- 너무 크거나 너무 작지 않게 적절한 단위로 분리한다.

### 5. 이슈 생성 확인 및 실행
생성할 이슈 제목과 본문을 사용자에게 보여주고 확인을 받는다.

확인 후 실행:
```bash
gh issue create --title "[TYPE] 한글 제목" --body "$(cat <<'EOF'
## Todo
- [ ] 구현 항목 1
- [ ] 구현 항목 2
EOF
)"
```

- `gh` CLI가 설치되어 있지 않으면 안내한다:
  > ⚠️ `gh` CLI가 필요합니다. `brew install gh` 로 설치 후 `gh auth login` 으로 인증하세요.

- 이슈 생성 완료 후 이슈 URL과 이슈 번호를 출력한다.
- 이어서 작업을 시작하려면 `start` 스킬을 실행하도록 안내한다.

## 출력 예시

```
📋 생성할 이슈 내용:

제목: [FEAT] 하이브리드 검색 API 구현

본문:
  ## Todo
  - [ ] ko-sroberta 벡터 검색 엔드포인트 구현
  - [ ] pg_bigm 키워드 검색 구현
  - [ ] RRF 결합 로직 구현
  - [ ] 검색 결과 200자 절삭 처리

이슈를 생성할까요?
```

```
✅ 이슈 생성 완료!
이슈 번호: #3
URL: https://github.com/...

작업을 시작하려면 start 스킬을 실행하세요.
```
