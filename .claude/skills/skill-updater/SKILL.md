---
name: skill-updater
description: 기존 skill 이름과 원하는 결과물 파일 위치를 입력받아, 결과물을 정확히 생성할 수 있도록 SKILL.md를 점검하고 자동 수정한다. 컨벤션 정합성 검증과 갭 분석을 포함한다.
---

기존 skill의 SKILL.md를 원하는 결과물 기준으로 점검하고 수정한다.

## 트리거 입력

사용자가 아래를 제공한 경우 실행한다.

- **필수**: 업데이트할 skill 이름 (예: `commit`, `issue`, `docs-sync`)
- **필수**: 원하는 결과물 파일 경로 또는 결과물에 대한 설명
- **선택**: 구체적인 수정 요구사항 (없으면 자동 분석)

입력이 부족하면 아래 순서로 질문한다:

1. > 어떤 skill을 업데이트할까요? (skill 이름을 입력하세요)
2. > 이 skill이 만들어내야 하는 결과물 파일의 위치 또는 결과물에 대한 설명을 알려주세요.

## 동작 순서

### 1. Skill 존재 확인

`.claude/skills/<skill-name>/SKILL.md` 파일이 존재하는지 확인한다.

- 존재하지 않으면 **중단**:
  > skill `<skill-name>`을 찾을 수 없습니다.
  > 사용 가능한 skill 목록: `ls .claude/skills/`

- 존재하면 SKILL.md 전체 내용을 읽는다.

### 2. 결과물 분석

사용자가 제공한 결과물 경로/설명을 기반으로 **기대하는 결과물의 구조와 내용**을 파악한다.

**경로가 주어진 경우:**
- 해당 파일이 이미 존재하면 내용을 읽어 구조, 포맷, 섹션 구성을 분석한다.
- 존재하지 않으면 경로의 확장자와 디렉토리 맥락에서 기대 포맷을 추론한다.
  - `.md` → 마크다운 문서, 섹션 구조
  - `.json` → JSON 스키마, 필드 구성
  - `.py`, `.ts` 등 → 코드 파일, 함수/클래스 구조

**설명이 주어진 경우:**
- 설명에서 기대하는 산출물의 형태, 포맷, 필수 포함 항목을 추출한다.

### 3. 갭 분석 (Gap Analysis)

기존 SKILL.md의 동작 순서가 **원하는 결과물을 정확히 만들어낼 수 있는지** 점검한다.

점검 항목:

| 점검 영역 | 확인 내용 |
|-----------|----------|
| 산출물 정의 | 결과물 파일 경로, 포맷, 필수 섹션이 명시되어 있는가 |
| 입력 정의 | 필요한 입력값이 빠짐없이 정의되어 있는가 |
| 동작 순서 | 결과물 생성에 필요한 모든 단계가 포함되어 있는가 |
| 엣지 케이스 | 파일 미존재, 입력 누락, 오류 상황 처리가 있는가 |
| 출력 예시 | 실제 기대 결과물과 일치하는 예시가 있는가 |
| 완료 조건 | 결과물이 올바르게 생성되었는지 검증 기준이 있는가 |

각 항목에 대해 **합격/불합격/개선 필요**로 판정한다.

### 4. 컨벤션 정합성 검증

다른 skill들의 공통 컨벤션과 비교하여 포맷 불일치를 점검한다.

**필수 컨벤션:**
- frontmatter에 `name`과 `description` 필드가 있는가
- `## 동작 순서` 섹션이 있는가
- 각 단계가 `### N. 제목` 형식인가
- 에러/중단 상황에 안내 메시지가 있는가
- 사용자 확인이 필요한 시점에 확인 절차가 있는가

**선택 컨벤션 (있으면 좋음):**
- `## 출력 예시` 섹션
- `## 완료 조건` 섹션
- `## 트리거 입력` 섹션

### 5. 수정 계획 제시

갭 분석과 컨벤션 검증 결과를 종합하여 **수정 계획**을 사용자에게 보여준다.

출력 형식:
```
== Skill 점검 결과: <skill-name> ==

[산출물 분석]
- 기대 결과물: <경로 또는 설명>
- 현재 SKILL.md 산출물 정의: <있음/없음/불일치>

[갭 분석 결과]
  합격: N건
  개선 필요: N건
  불합격: N건

[수정 항목]
1. <영역> — <현재 상태> → <수정 내용>
2. <영역> — <현재 상태> → <수정 내용>
...

[컨벤션 불일치]
- <항목>: <현재> → <권장>

이대로 수정할까요?
```

### 6. 사용자 확인 후 수정 실행

사용자가 확인하면 SKILL.md를 수정한다.

**수정 원칙:**
- 기존 내용 중 유효한 부분은 최대한 보존한다.
- 추가/변경이 필요한 부분만 수정한다.
- frontmatter의 `description`은 수정된 동작을 반영하여 갱신한다.
- 수정 후 전체 문서가 자연스럽게 읽히는지 확인한다.

**수정 불가 항목:**
- `name` 필드는 변경하지 않는다 (디렉토리명과 일치해야 함).
- 다른 skill 파일은 건드리지 않는다.

### 7. 수정 결과 검증

수정 완료 후 최종 검증을 수행한다.

- 수정된 SKILL.md를 다시 읽어 갭 분석 6개 항목을 재점검한다.
- 컨벤션 필수 항목이 모두 충족되는지 확인한다.
- 문제가 남아있으면 추가 수정 여부를 묻는다.

---

## 출력 예시 (갭 발견)

```
== Skill 점검 결과: api-gen ==

[산출물 분석]
- 기대 결과물: docs/6. API 명세서/endpoints/<method>-<path>.md
- 현재 SKILL.md 산출물 정의: 경로만 있고 필수 섹션 정의 없음

[갭 분석 결과]
  합격: 3건 (입력 정의, 동작 순서, 엣지 케이스)
  개선 필요: 2건 (산출물 정의, 출력 예시)
  불합격: 1건 (완료 조건 누락)

[수정 항목]
1. 산출물 정의 — 경로만 명시 → 필수 섹션(요청/응답/에러코드) 추가
2. 출력 예시 — 간략한 예시 → 실제 결과물과 동일한 전체 예시로 교체
3. 완료 조건 — 없음 → 파일 생성 확인 + 필수 섹션 포함 여부 검증 추가

[컨벤션 불일치]
- 없음

이대로 수정할까요?
```

## 출력 예시 (문제 없음)

```
== Skill 점검 결과: commit ==

[산출물 분석]
- 기대 결과물: git commit 실행 (파일 산출물 없음)
- 현재 SKILL.md 산출물 정의: 적절함

[갭 분석 결과]
  합격: 6건
  개선 필요: 0건
  불합격: 0건

[컨벤션 불일치]
- 없음

수정할 항목이 없습니다. 현재 SKILL.md가 기대 결과물을 정확히 생성할 수 있습니다.
```

---

## 완료 조건

- 갭 분석 6개 항목이 모두 합격 또는 개선 완료됨
- 컨벤션 필수 항목이 모두 충족됨
- 수정된 SKILL.md가 원하는 결과물을 만들어낼 수 있는 동작 순서를 포함함
- `description` 필드가 수정된 동작을 정확히 반영함
- 사용자가 수정 내용을 확인함
