# 호롱호롱 사용법

호롱호롱은 메뉴바에서 타이머, 메모, 사용 시간 통계, 뉴스 큐레이션, Agent 실험을 빠르게 실행하는 macOS 앱입니다.

타이머, 메모, 통계는 앱을 실행하면 바로 사용할 수 있습니다. 뉴스와 Agent는 외부 AI CLI를 호출하므로, 해당 기능을 쓰기 전에 사용하는 CLI가 터미널에서 실행되는지만 확인하면 됩니다.

## 0. 처음 준비

뉴스와 Agent 기능 중 하나라도 쓰려면 아래 CLI 중 필요한 것을 설치합니다.

```bash
# 뉴스: claude, codex, gemini, opencode 중 하나 이상
npm install -g @anthropic-ai/claude-code
npm install -g @openai/codex
npm install -g @google/gemini-cli
npm install -g opencode-ai
```

설치 후 터미널에서 사용할 명령이 동작하는지 확인합니다.

```bash
claude --version
codex --version
gemini --version
opencode --version
```

소스에서 빌드한 경우 뉴스 파이프라인 의존성도 한 번 준비합니다.

```bash
cd Agents/news_report
uv sync
```

Agent 기능은 Terminal 앱을 열어 CLI 명령을 실행합니다. 처음 사용할 때 macOS가 자동화 권한을 묻는다면 허용합니다. 거부했다면 **System Settings → Privacy & Security → Automation**에서 호롱호롱의 Terminal 제어를 허용합니다.

## 1. 메뉴바 팝오버

메뉴바 아이콘을 누르면 팝오버가 열립니다. 상단 탭에서 **타이머**, **메모**, **통계**, **뉴스**, **Agent**를 전환하고, 하단의 **설정** 버튼으로 별도 설정 창을 엽니다. 설정 창은 `⌘,`로도 열 수 있습니다.

## 2. 뉴스 탭

뉴스 탭은 선택한 AI CLI로 관심 정보 리포트를 만듭니다. 기본 흐름은 단순합니다.

1. 메뉴바 아이콘을 누르고 **뉴스** 탭을 엽니다.
2. 오른쪽 위 `Provider`에서 사용할 CLI를 고릅니다.
3. **리포트 생성**을 누릅니다.
4. 실행 중에는 수집, 정규화, 중복 제거, 분류, 랭킹, 요약, 렌더링 진행 상태를 확인합니다.
5. 완료되면 **최근 리포트** 목록에서 리포트를 클릭해 마크다운 파일을 엽니다.

처음에는 별도 설정 없이도 기본 소스로 실행할 수 있습니다. 수집 대상을 직접 정하고 싶을 때만 **설정 → 뉴스**로 이동합니다.

### 뉴스 설정에서 바꾸는 것

- **소스**: YouTube 채널, YouTube 재생목록, Google News, YOZM IT, Hacker News, RSS를 추가하거나 끕니다.
- **관심 키워드**: 뉴스 수집과 관련도 필터링에 사용할 키워드를 칩으로 관리합니다. Agent 관심사와는 별개입니다.
- **요약 에이전트**: 기본 Provider를 `claude`, `codex`, `gemini`, `opencode` 중에서 고릅니다.
- **소스당 최대 항목 수**: 한 번에 가져올 기사 수를 조정합니다.
- **일일 리포트 저장 위치**: 마크다운 리포트, 메타데이터, 로그가 저장될 위치를 바꿉니다.

리포트는 저장 위치 아래에 다음처럼 쌓입니다.

```text
data/reports/   # 마크다운 리포트
data/meta/      # 실행 결과 메타데이터
data/logs/      # 파이프라인 로그
```

### 뉴스 문제가 날 때

- `uv` 또는 `Python 3` 관련 안내가 뜨면 `uv sync`와 Python 설치 상태를 확인합니다.
- `claude: command not found` 같은 오류가 나면 선택한 Provider 명령이 터미널에서 실행되는지 확인합니다.
- 리포트가 너무 넓거나 엉뚱하면 뉴스 설정에서 소스와 관심 키워드를 줄입니다.

## 3. Agent 탭

Agent 탭은 관심사와 아이디어 파일을 바탕으로 며칠짜리 실험 계획을 만들고, 오늘 할 부분만 다시 실행하도록 돕습니다.

### 한 번 설정하기

**설정 → AI Agent**에서 아래 항목을 정합니다.

- **실험 루트 폴더**: 앱이 이 폴더 아래에 `ideas`와 `outputs`를 자동으로 만듭니다.
- **기본 Agent**: `Codex`, `Claude`, `Gemini` 중 하나를 고릅니다.
- **계획 일수**: 한 번에 만들 실험 계획 기간을 정합니다.
- **관심사**: 실험 계획에 반영할 키워드를 칩으로 관리합니다. 뉴스 관심 키워드와는 별개입니다.

폴더 구조는 다음과 같습니다.

```text
Agents/experiments/
├── ideas/     # 아이디어, 메모, 참고 마크다운
└── outputs/   # 생성된 실험 계획과 실행 결과
```

### 계획 만들기

1. 참고할 메모나 아이디어가 있으면 `ideas` 폴더에 넣습니다.
2. 메뉴바에서 **Agent** 탭을 엽니다.
3. Agent와 계획 일수를 확인합니다.
4. **계획 생성**을 누릅니다.

Terminal 앱이 열리고 선택한 CLI가 실행됩니다. 계획 파일은 `outputs` 폴더에 `YYYY-MM-DD_experiment_plan_Nd.md` 형식으로 저장됩니다.

### 오늘 실험 실행하기

1. **오늘 실험 실행**을 누릅니다.
2. 앱이 `outputs`에서 가장 최근 계획 파일을 찾습니다.
3. 오늘 날짜 섹션만 추출해 선택한 CLI에 전달합니다.

오늘 날짜 섹션이 없는 계획 파일만 있으면 실행할 수 없습니다. 그 경우 새 계획을 생성하거나 계획 파일에 오늘 날짜 섹션을 추가합니다.

## 4. 타이머 탭

타이머 탭은 집중과 휴식 사이클을 관리합니다.

- 기본 프리셋은 `포모도로(50/5)`, `긴 집중(100/10)`, `커스텀(60/10)`입니다.
- 집중 카테고리를 선택하면 완료된 집중 세션이 해당 카테고리 통계에 기록됩니다.
- 타이머가 끝나면 시스템 알림과 앱 내부 토스트가 표시됩니다.
- 프리셋 시간, 메뉴바 표시 형식, 시간 표시 방식은 **설정 → 타이머**에서 바꿉니다.

## 5. 메모 탭

메모 탭은 짧은 메모를 저장하고 최신순으로 확인하는 공간입니다.

- 기본 퀵 메모 단축키는 `⌘⇧N`입니다.
- 단축키를 누르면 어느 앱에서든 플로팅 메모 입력창이 열립니다.
- 저장된 메모는 로컬 SwiftData에 보관됩니다.
- 퀵 메모 단축키는 **설정 → 단축키**에서 바꿀 수 있습니다.

## 6. 통계 탭

통계 탭은 앱 사용 시간을 카테고리별로 요약합니다.

- 팝오버에서는 **오늘**과 **이번 주** 요약을 전환해 볼 수 있습니다.
- **상세 보기**를 누르면 일간, 주간, 월간 통계 창이 열립니다.
- 일간 상세에서는 타임라인, 카테고리 분포, 앱별 사용량, 포모도로 기록을 확인합니다.
- 일간 상세의 **편집** 버튼으로 앱 사용 세그먼트를 수동 추가, 수정, 삭제할 수 있습니다.
- Chrome/Safari는 현재 URL을 읽어 YouTube/Netflix는 `엔터`, 검색·문서·기술 블로그 계열은 `조사`로 분류합니다.

통계 동작은 **설정 → 카테고리 매핑**과 **설정 → 통계**에서 조정합니다.

- 앱별 카테고리 규칙 추가·수정·삭제
- 카테고리 이름과 이모지 변경
- 자리 비움 감지 임계값 조정
- 같이 쓰는 카테고리 쌍 등록
- 타임라인 표시 시간과 버킷 간격 조정
- 휴가 기간 등록

휴가 기간으로 등록한 날짜에는 기록이 중단되고, 통계 상세 화면에서 해당 날짜가 휴가로 표시됩니다.

## 7. 설정 창

설정 창은 앱의 기본 동작을 조정하는 곳입니다. 사이드바 검색으로 필요한 항목을 찾을 수 있고, 하단의 **기본값으로 복원**은 현재 보고 있는 설정 페이지만 초기화합니다.

주요 페이지는 다음과 같습니다.

| 페이지 | 용도 |
|------|------|
| 일반 | 앱 기본 동작 |
| 외관 | 화면 모드, 팝오버 테마 |
| 타이머 | 프리셋 시간, 메뉴바 표시 |
| 단축키 | 퀵 메모 단축키 |
| 카테고리 매핑 | 카테고리, 앱 규칙, 자리 비움 기준 |
| 통계 | 타임라인 표시, 휴가 기간 |
| 뉴스 | 뉴스 소스, 관심 키워드, Provider, 저장 위치 |
| AI Agent | 실험 루트 폴더, Agent, 계획 일수, 관심사 |
| 메모 | 퀵 메모 동작 |
| 데이터 | 데이터 위치와 백업 관련 항목 |
| 정보 | 버전, GitHub, 라이선스 |

## 8. 데이터 저장 위치

앱 데이터는 로컬 SwiftData 저장소와 사용자가 지정한 폴더에 저장됩니다.

- 타이머, 메모, 사용 시간 통계, 통계 캐시, 카테고리 규칙: 로컬 SwiftData 저장소
- 뉴스 리포트: **설정 → 뉴스 → 일일 리포트 저장 위치**
- Agent 실험 계획과 결과물: **설정 → AI Agent → 실험 루트 폴더** 아래 `outputs`
- Agent 아이디어 파일: **설정 → AI Agent → 실험 루트 폴더** 아래 `ideas`

앱을 삭제해도 macOS의 앱 데이터 컨테이너나 사용자가 지정한 저장 폴더를 따로 지우지 않으면 일부 데이터가 남을 수 있습니다.
