# 호롱호롱 사용법

호롱호롱은 메뉴바에서 타이머, 메모, 사용 시간 통계, 뉴스 큐레이션, Agent 실험을 빠르게 실행하는 macOS 앱입니다.

뉴스 탭과 Agent 탭은 외부 AI CLI를 호출하므로 먼저 환경을 준비해야 합니다. 타이머, 메모, 통계, 설정 탭은 앱을 실행하면 바로 사용할 수 있습니다.

## 0. 뉴스 / Agent 기능 준비

뉴스 탭과 Agent 탭 중 하나를 쓰려면 AI CLI가 최소 하나 이상 터미널에서 동작해야 합니다.

```bash
# 필요한 CLI 중 하나 이상 설치
npm install -g @anthropic-ai/claude-code   # claude
npm install -g @openai/codex               # codex
npm install -g @google/gemini-cli          # gemini
npm install -g opencode-ai                 # opencode
```

설치 후 터미널에서 `claude --version`, `codex --version`, `gemini --version`, `opencode --version` 중 사용하는 명령이 동작하는지 확인합니다.

뉴스 파이프라인은 Python 의존성을 사용합니다. 소스에서 빌드한 경우 최초 1회만 아래 명령을 실행합니다.

```bash
cd Agents/news_report
uv sync
```

## 1. 뉴스 탭

뉴스 탭은 YouTube, Google News, 요즘IT에서 항목을 수집하고, 선택한 AI CLI로 관심사 기준 리포트를 만듭니다.

### 기본 동작

1. 메뉴바 아이콘을 클릭하고 **뉴스** 탭을 엽니다.
2. `Provider`에서 사용할 AI CLI를 선택합니다.
3. 필요하면 `리포트 저장 경로`, `관심사 키워드`, `YouTube 채널`을 조정합니다.
4. **리포트 생성**을 누릅니다.
5. 완료되면 최근 리포트 목록에서 문서 아이콘을 눌러 마크다운 리포트를 엽니다.

리포트는 기본적으로 아래 위치에 저장됩니다.

```text
Agents/news_report/data/reports/YYYY-MM-DD.md
```

앱에서 `리포트 저장 경로`를 바꾸면 선택한 폴더 아래의 `data/reports`, `data/meta`, `data/logs`에 결과가 쌓입니다.

### 설정 항목

| 항목 | 설명 |
|------|------|
| `Runner` | 앱이 자동 감지한 `Agents/news_report/runner.py` 경로입니다. 현재 UI에서는 직접 변경하지 않습니다. |
| `리포트 저장 경로` | 리포트, 메타데이터, 로그가 저장될 루트 폴더입니다. |
| `Provider` | `claude`, `codex`, `gemini`, `opencode` 중 실행할 AI CLI입니다. |
| `관심사 키워드` | 리포트 필터링·랭킹·요약 기준입니다. 기본값은 `AI, 개발, 생산성, 자동화`입니다. |
| `YouTube 채널` | 채널 ID 또는 채널 URL을 추가할 수 있습니다. 비어 있으면 기본 IT 채널이 사용됩니다. |

### 수집 소스 기본값

| 소스 | 기본 상태 | 기본값 |
|------|----------|--------|
| YouTube | 켜짐 | Google Developers |
| Google News | 켜짐 | `AI`, `개발`, `생산성`, `자동화` |
| 요즘IT | 켜짐 | `개발`, `생산성`, `AI`, `자동화` |
| LinkedIn | 꺼짐 | 인증이 필요한 소스라 기본 비활성화 |

### 자주 나는 문제

- `ModuleNotFoundError`: `Agents/news_report`에서 `uv sync`를 실행했는지 확인합니다.
- `claude: command not found` 같은 오류: 선택한 Provider의 CLI가 터미널에서 실행 가능한지 확인합니다.
- `Runner 경로를 설정해주세요`: 소스 폴더 구조가 바뀌었거나 앱이 `Agents/news_report/runner.py`를 자동 감지하지 못한 상태입니다. 소스 루트에서 앱을 다시 빌드해 실행합니다.

## 2. Agent 탭

Agent 탭은 관심사와 아이디어 파일을 바탕으로 N일짜리 실험 계획을 만들고, 오늘 분량을 선택한 AI CLI에 전달해 실행합니다.

### 실험 루트 폴더

Agent 탭에서는 폴더를 하나만 고릅니다. 앱은 그 루트 아래를 자동으로 나눠 사용합니다.

```text
Agents/experiments/
├── ideas/    # 아이디어, 메모, 참고 마크다운
└── outputs/  # 생성된 실험 계획과 결과물
```

`실험 루트 폴더`를 변경하면 `ideas`와 `outputs` 경로도 함께 바뀝니다.

### 계획 생성

1. **Agent** 탭을 엽니다.
2. `실험 루트 폴더`를 확인하거나 변경합니다.
3. `Agent`에서 `Codex`, `Claude`, `Gemini` 중 하나를 선택합니다.
4. `계획 일수`와 `관심사 키워드`를 정합니다. 기본 관심사는 `생산성, 자동화, 학습`입니다.
5. **계획 생성**을 누릅니다.

계획 생성이 시작되면 Terminal 앱이 열리고 선택한 AI CLI가 실행됩니다. 결과는 `outputs` 폴더에 `YYYY-MM-DD_experiment_plan_Nd.md` 형태로 저장됩니다.

### 오늘 실험 실행

1. **오늘 실험 실행**을 누릅니다.
2. 앱이 `outputs` 폴더에서 가장 최근의 `_experiment_plan_` 파일을 찾습니다.
3. 오늘 날짜에 해당하는 섹션만 추출해 선택한 AI CLI에 전달합니다.

처음 실행할 때 macOS가 Terminal 자동화 권한을 물어볼 수 있습니다. 거부했다면 **System Settings → Privacy & Security → Automation**에서 호롱호롱의 Terminal 제어를 허용합니다.

## 3. 타이머 탭

타이머 탭은 포모도로 집중·휴식 사이클을 관리합니다.

- 기본 프리셋은 `포모도로(50/5)`, `긴 집중(100/10)`, `커스텀`입니다.
- 커스텀 기본값은 `집중 60분 / 휴식 10분`입니다.
- 집중 카테고리를 선택하면 완료된 집중 세션이 해당 카테고리로 기록됩니다.
- 타이머가 끝나면 시스템 알림과 앱 내부 토스트가 표시됩니다.

## 4. 메모 탭

메모 탭은 짧은 메모를 저장하고 최신순으로 확인하는 공간입니다.

- 글로벌 단축키로 플로팅 메모 입력창을 열 수 있습니다.
- 저장된 메모는 로컬 SwiftData에 보관됩니다.
- 메모 탭에서 최근 메모를 바로 확인할 수 있습니다.

## 5. 통계 탭

통계 탭은 오늘의 앱 사용 시간을 카테고리별로 요약합니다.

- 앱은 활성 앱을 백그라운드에서 추적합니다.
- 기본 카테고리는 `업무`, `개발`, `공부`, `조사`, `기록`, `소통`, `엔터`, `기타`입니다.
- Chrome/Safari는 현재 URL을 읽어 YouTube/Netflix는 `엔터`, 검색·문서·기술 블로그 계열은 `조사`, 그 외는 `기타`로 분류합니다.
- **상세 보기**를 누르면 일간, 주간, 월간 통계를 확인할 수 있습니다.

## 6. 설정 탭

설정 탭에서는 앱의 기본 동작을 조정합니다.

- 앱별 카테고리 매핑 규칙을 추가, 수정, 삭제할 수 있습니다.
- 카테고리 이름과 이모지를 바꿀 수 있습니다.
- 카테고리를 삭제하면 기존 기록은 `기타`로 이동합니다.
- 유휴 시간 기준, 타이머 프리셋, 관심사 키워드 등을 조정할 수 있습니다.

## 7. 저장 위치

앱 데이터는 기본적으로 로컬 SwiftData 저장소와 사용자가 지정한 폴더에 저장됩니다.

- 타이머, 메모, 사용 시간 통계, 카테고리 규칙: 앱의 로컬 SwiftData 저장소
- 뉴스 리포트: 뉴스 탭의 `리포트 저장 경로`
- Agent 실험 계획: Agent 탭의 `실험 루트 폴더/outputs`
- Agent 아이디어 파일: Agent 탭의 `실험 루트 폴더/ideas`

앱을 삭제해도 macOS의 앱 데이터 컨테이너나 사용자가 지정한 저장 폴더를 따로 지우지 않으면 일부 데이터가 남을 수 있습니다.
